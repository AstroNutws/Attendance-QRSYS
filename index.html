05Frequently Asked Questions
FAQ

Questions & Answers

What are the four roles in Project QrSys?▼
Normal User — scans QR codes and views their own attendance. Admin — handles daily operations, scans for users, manages sections. Super Admin — creates admin accounts and manages credentials. Owner — holds the database with full system authority.
What exactly can a Normal User do?▼
Normal Users can scan a QR code to mark their attendance and view their own records. They can submit a request to update personal information — but this change requires Admin approval before taking effect.
Can Admins scan QR codes for users?▼
Yes. Admins can scan QR codes on behalf of users — useful when a user cannot scan themselves. Admins also manage sections, approve info change requests, and have full Admin Panel access.
What is the difference between Admin and Super Admin?▼
Admins handle day-to-day operations. Super Admins go higher: they create new admin accounts, modify peer credentials, reset passwords, and delegate Super Admin status via the dedicated Super Admin Panel.
Who has direct access to the database?▼
Only the Owner has database-level authority — exclusively through their private Supabase Dashboard. Normal Users, Admins, and Super Admins have zero direct database access.
